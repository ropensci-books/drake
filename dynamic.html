Chapter 5 Dynamic branching

5.1 Motivation

With static branching (explained in the following chapter) we can concisely create plans with large numbers of targets. However, static branching has major issues.

  1. If a plan gets too large, functions drake_config() and outdated() become very slow, which severely slows down the initialization of make().
  2. We need to declare every single target in advance. We cannot define new targets based the values returned from other targets.
  3. The graph visualizations are too slow, too cumbersome, and too unresponsive when the number of static targets is large.
  4. static branching relies on metaprogramming and code manipulation, which makes it difficult to use and understand.

Dynamic branching, supported in versions above 7.7.0, solves these problems.

5.2 Which kind of branching should I use?

With dynamic branching, make() is faster to initialize, and you have far more flexibility. With static branching, you have meaningful target names, and it is easier to predict what the plan is going to do in advance. There is a ton of room for overlap and personal judgement, and you can even use both kinds of branching together!

library(drake)

plan <- drake_plan(
  locations = get_location_db(), # many rows
  coord = target(
    geocode_location(locations, method),
    dynamic = map(locations),
    transform = map(method = c("address", "postal", "boundary")) #static
  ),
  results = target(
    bind_rows(coord),
    dynamic = combine(coord),
    transform = combine(coord)
  )
)

plot(plan)

5.3 Dynamic targets

A dynamic target has multiple sub-targets. Prior to running make(), we do not know how many sub-targets there will be, nor what they will contain. This flexibility lets the data drive the plan. For example, we can fit a regression model to each continent in Gapminder data and give each model its own target. To activate dynamic branching, use the dynamic argument of target().

library(broom)
library(drake)
library(gapminder)
library(tidyverse)

fit_model <- function(dataset, continent) {
  dataset %>%
    filter(continent == !!continent) %>% # The !! is important.
    lm(formula = gdpPercap ~ year) %>%
    tidy() %>%
    mutate(continent = !!continent)
}

plan <- drake_plan(
  # This dataset can change, and we want
  # the downstream targets to update.
  dataset = gapminder %>%
    mutate(gdpPercap = scale(gdpPercap)),
  
  # We need a grouping variable.
  continent = unique(dataset$continent),
  
  # Fit GDP vs year for each country.
  model = target(
    fit_model(dataset, continent),
    dynamic = map(continent) # Activate dynamic branching!
  )
)

make(plan)
#> target dataset
#> target continent
#> subtarget model_b0cb11b5
#> subtarget model_f3eaf4f9
#> subtarget model_ffd4e544
#> subtarget model_d2e01bcc
#> subtarget model_ff9e33d1

The sub-targets have strange names (there are good reasons!) but you do not need sub-target names in order to fetch values.

readd(model, subtargets = c(1, 2))

To select specific targets, simply load the original grouping variable and select the indices you need.

loadd(continent)

vis_drake_graph(config)

5.4 map()

The dynamic map() transformation creates a new sub-target for each element of the grouping variables you supply.

plan <- drake_plan(
  static_numbers = seq_len(2),
  static_letters = c("a", "b"),
  dynamic_lowercase = target(
    paste0(static_numbers, static_letters),
    dynamic = map(static_numbers, static_letters)
  ),
  dynamic_uppercase = target(
    toupper(dynamic_lowercase),
    dynamic = map(dynamic_lowercase)
  )
)

make(plan)
#> target static_numbers
#> target static_letters
#> subtarget dynamic_lowercase_13620767
#> subtarget dynamic_lowercase_ff3dee4f
#> subtarget dynamic_uppercase_1e011b6d
#> subtarget dynamic_uppercase_4ef40845

For grouping variables that are vectors and simple lists, dynamic branching iterates over the elements.

readd(dynamic_lowercase)
#> [[1]]
#> [1] "1a"
#> 
#> [[2]]
#> [1] "2b"

readd(dynamic_uppercase)
#> [[1]]
#> [1] "1A"
#> 
#> [[2]]
#> [1] "2B"

For grouping variables that are array-like objects (anything with a non-null dim()) dynamic branching iterates over the rows. This includes not only arrays and matrices, but data frames and tibbles as well.1

plan <- drake_plan(
  dataset = head(gapminder, n = 3),
  row = target(dataset, dynamic = map(dataset))
)

make(plan)

Regardless of the data type of object x, NROW(x) tells you how many sub-targets you will get from map(x).

NROW(letters)
#> [1] 26

NROW(mtcars)
#> [1] 32

5.5 cross()

cross() is like map() except we create a new target for each combination of grouping variables.

plan <- drake_plan(
  numbers = seq_len(2),
  letters = c("a", "b"),
  result = target(
    c(numbers, letters),
    dynamic = cross(numbers, letters)
  )
)

make(plan)
#> target numbers
#> target letters
#> subtarget result_13620767
#> subtarget result_fcae20ae
#> subtarget result_461b9aa9
#> subtarget result_ff3dee4f

readd(result)

#> [1] "2" "a"

5.6 combine()

combine() can group together sub-targets or split up static targets. The .by argument lets us control the aggregation. Let’s fit a model to each continent in the Gapminder dataset and then combine all the results at the end.

fit_model <- function(dataset) {
  dataset %>%
    lm(formula = gdpPercap ~ year) %>%
    tidy() %>%
    mutate(continent = dataset$continent[1])
}
  
plan <- drake_plan(
  # Let's fit a model for each continent and then
  # combine the results at the end.
  dataset = gapminder %>%
    mutate(gdpPercap = scale(gdpPercap)),
  
  # We need a target to act as a grouping variable.
  continent = dataset$continent,
  
  # Fit a model for each continent.
  model = target(
    fit_model(dataset),
    dynamic = combine(dataset, .by = continent)
  ),
  
  # Aggregate the results together.
  results = target(
    bind_rows(model),
    dynamic = combine(model) # no .by necessary
  )
)

make(plan)

#> target continent

#> subtarget model_835a0a23
